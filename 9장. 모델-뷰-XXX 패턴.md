1. 정의
   - 모델과 뷰로 이루어진 패턴이다.
2. 클래스 다이어그램
   - ![모델뷰 패턴 클래스다이어그램](./img/modelView_pattern_uml.png)
3. 구성요소
   - Model
     - 애플리케이션에서 사용되는 데이터와 그 데이터를 처리한다.
   - View
     - 사용자에게 보여지는 UI 부분이다.
     - 화면에 Model이 가진 데이터를 보여주는 역할을 한다.
4. 특징
   - 입력받은 데이터를 뷰에서 직접 처리한다.
5. 장점
   - 단순해서 로직 처리를 직관적으로 할 수 있다.
6. 단점
   - 복잡한 로직 처리를 해야 하는 애플리케이션에는 적합하지 않다.
